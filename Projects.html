Gladiator Game

Gladiator was made during my final year at Algoma University over an 8 month period. It was built using the Unity engine with all modeled assets being made in Blender including rigging and animation. Our original goal was to create a basic video game using Agile Methodologies with simple assets so that we can learn both a new language C# and how a game engine works. During the second semester of development, once we got the basics done we started adding in simple AI, multiplayer and cleaner assets.

The world was set in a post apocalyptic area and focused primarily on melee and ranged combat. We designed an arena waved based game mode to demonstrate our mechanics and AI. The rounds would start with only a couple of enemies and players would quickly be able to eliminate them using various attack types. A stab would apply a bleed while an overhead attack would do increased damage while leaving the player vulnerable to attack. A slash attack was also included which would be able to hit multiple enemies. As each round progressed the difficulty was increased as more enemies would spawn during each wave.

For the most part we accomplished what we set out to do however some areas we would have liked to improve or add are:

  • Improved AI
  • Difficulty Settings besides increasing the number of enemies
  • Smoother Animations
  • Improved Environments
  • Additional Working Weapons
  • Classes
  • Bosses

Music Metadata Fixer

I enjoy listening to music while I work, surf the web or while playing games. As a result I felt like taking on a small project and designing a music library of my own. It originally started due to many of my music files missing meta-data such as artist, year, genre, among many others. Currently the program is able to extract the meta-data details and print them to the screen. Additional information about this project will be posted as more is developed.
